# 项目概览

## 仓库定位

`PenpotWorkflow` 是一个围绕 Penpot 展开的工作流仓库，目标是为 Godot 游戏项目提供一套清晰、开源、可复用的 UI 原型协作方式。

它主要回答三个问题：

- 用什么工具做原型
- 原型怎样交付给 Godot 项目
- 团队内部怎样用统一约定减少返工

## 为什么选择 Penpot

Penpot 的价值主要在于：

- 开源，可自托管
- 设计稿、组件、页面结构都可以协作维护
- 对小团队较友好，不依赖商业订阅
- 适合作为流程中“讨论设计意图”的中间层

对于游戏项目来说，Penpot 不是最终 UI 技术方案，而是原型和交付方案的一部分。

## 适合解决的问题

- 线框图阶段快速比对多个界面布局方案
- 把复杂 HUD、面板、弹窗拆成可复用组件
- 将视觉稿中的状态变化用统一页面和命名方式记录下来
- 给 Godot 实现阶段提供更明确的尺寸、层级和资源说明

## 不适合解决的问题

- 直接生成可运行的 Godot UI
- 替代 Godot 中的动画、输入、响应式布局和实际逻辑
- 通过一个外部设计工具完整表达所有运行时细节

## 推荐工作边界

Penpot 负责：

- 页面草图
- 状态稿
- 组件定义
- 资源导出说明
- 设计评审材料

Godot 负责：

- `Control` 节点树搭建
- 运行时交互逻辑
- 输入适配
- 动画与过渡
- 多分辨率适配
- 性能与可用性调优

## 推荐演进路径

1. 先写清文档和命名规范
2. 再沉淀页面模板和组件模板
3. 然后为常见导出动作补脚本
4. 最后才考虑半自动化映射到 Godot 的辅助工具

这个顺序的原因很直接：在流程尚不稳定时，自动化只会固化混乱。先统一语言和交付物，再考虑提效。
